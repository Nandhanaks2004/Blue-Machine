Section 03

Reconnaissance Phase

The first step in any penetration test is understanding your target. Reconnaissance involves gathering information about the target system, network, and potential attack vectors before launching any exploits.

What is Reconnaissance?

In the context of the Blue machine, reconnaissance involves identifying that the target is a Windows 7 system, discovering its IP address, and understanding what services it is running. This phase sets the foundation for all subsequent exploitation steps.

A

Using ping sweeps and network scans to identify live hosts on the target subnet before focusing on specific machines.

O

Open Port Detection

Scanning for open ports to understand which services are accessible, revealing potential entry points for exploitation.

S

Service Fingerprinting

Determining exact service versions and configurations to identify vulnerable software and plan targeted attacks.

M

Attack Surface Mapping

Creating a comprehensive map of all potential attack vectors based on discovered services, versions, and configurations.

Section 05

Understanding SMB

Server Message Block (SMB) is a network file-sharing protocol used primarily by Windows systems. Understanding SMB is crucial because EternalBlue targets SMBv1, an older version of this protocol.

What is SMB?

SMB (Server Message Block) is a protocol used for file sharing, printer sharing, and inter-process communication on Windows networks. It operates on ports 139 (NetBIOS) and 445 (direct SMB). The EternalBlue exploit targets a vulnerability in SMBv1 that allows remote code execution without authentication.

Section 06

MS17-010 Vulnerability Analysis

MS17-010 is a critical remote code execution vulnerability in Microsoft's SMBv1 server. EternalBlue is the exploit that weaponizes this vulnerability, allowing attackers to execute arbitrary code on the target system.

How EternalBlue Works

1Attacker sends a crafted SMBv1 packet to the target on port 445
2The packet triggers a buffer overflow in the SMBv1 driver (srv.sys)
3Shellcode is injected into kernel memory space
4The shellcode executes with SYSTEM privileges bypassing security controls
5A reverse shell or Meterpreter payload is delivered to the attacker
Section 07

EternalBlue in the Real World

The EternalBlue exploit was infamously used in several major global ransomware attacks, causing billions of dollars in damages and affecting hundreds of thousands of systems worldwide.

2017

WannaCry Ransomware

EternalBlue was used to spread the WannaCry ransomware globally in May 2017. The attack affected over 200,000 computers across 150 countries, causing an estimated $4 billion in damages. Major organizations including the UK National Health Service (NHS) were crippled.

2017

NotPetya Attack

In June 2017, the NotPetya attack used EternalBlue alongside other exploits to spread rapidly across networks. Originally targeting Ukraine, it spread globally causing over $10 billion in damages and affecting companies like Maersk, Merck, and FedEx.

2017

BadRabbit Campaign

In October 2017, the BadRabbit ransomware campaign also utilized EternalBlue for propagation. While less widespread than WannaCry, it demonstrated that the exploit remained a persistent threat months after the patch was released.

$

Financial Impact

Combined damages from EternalBlue-based attacks exceeded $10 billion, affecting critical infrastructure, healthcare, and enterprise systems globally.

H

Healthcare Crippled

Hospitals and healthcare systems were forced to cancel surgeries and divert emergency patients, highlighting the real-world consequences of unpatched systems.

G

Global Propagation

The self-propagating nature of EternalBlue allowed attacks to spread across the internet in minutes without any user interaction, making containment extremely difficult.

Section 13

SYSTEM Privilege Understanding

SYSTEM is the highest privilege level on Windows, equivalent to root on Linux. Understanding privilege levels is crucial for comprehending the full impact of EternalBlue exploitation.

UserLimited access
â†‘
AdminElevated access
â†‘
SYSTEMComplete control
U

Unrestricted Access

SYSTEM privileges provide unrestricted access to all files, processes, and registry keys on the Windows operating system without any security restrictions.

K

Kernel-Level Control

SYSTEM operates at the kernel level, allowing interaction with hardware drivers, system calls, and low-level operating system functions that are inaccessible to standard users.

P

Persistence & Evasion

With SYSTEM privileges, attackers can install persistent backdoors, disable security software, and hide their presence from standard detection mechanisms.

Section 21

Defensive Security Measures

Understanding the attack is only half the battle. Implementing proper defensive measures prevents these attacks from succeeding in your environment.

!

Patch Management

Why: EternalBlue exploits unpatched SMBv1 vulnerabilities

Fix: Apply MS17-010 patch, disable SMBv1 via Group Policy

!

Network Segmentation

Why: Lateral movement spreads infections rapidly

Fix: Segment networks, restrict SMB access between zones

!

Endpoint Detection

Why: EDR detects exploit behavior in memory

Fix: Deploy EDR with behavioral detection rules

!

Firewall Rules

Why: Open ports 139/445 are the attack vector

Fix: Block SMB ports at perimeter firewalls

!

Least Privilege

Why: Limits damage from compromised accounts

Fix: Implement RBAC and regular privilege audits

!

Log Monitoring

Why: Attack leaves traces in event logs

Fix: Centralize logs, alert on SMB anomalies
